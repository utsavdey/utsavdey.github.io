The C++ I/O system


There are currently two versions of the C++ object oriented I/O library in use. The older one that is based upon the original specification for C++ & the newer one defined by standard C++.

The old I/O library is supported by the header file <iostream.h>.

The new I/O library is supported by the header <iostream>.

There are two main difference between the old & new C++ I/O libraries.

First, the new I/O libraries contains a few data types. Thus, the new I/O libraries is essentially a superset of old one.

Newly, all program originally written for the old library will compile without substantive changes when the new library is used.

Second the old-style I/O library was in the global namespace. The new-style library is in the std namespace.


C++ Streams


A stream is a logical device that either produces or consumes information. A stream is linked to a physical device by the I/O system. All streams behave in the same way even through the actual physical devices they are connected to many different substantially. Because all streams behave the same I/O functions can operate on virtually any type of physical device.

Example: - users can use the same function that writes to a file to write to the pointer on to the screen.

The advantage to this approach is that users need learn only one I/O system.

Standard Template Library (STL)

In order to help the C++ users in generic programming, Alexander Stepanov & Mong Lee of HP developed a set of general purpose templatized classes (data structure) function (algorithm) that could be used as a standard approach for storing and processing of data. The collection of these generic classes & functions in called the Standard Template library (STL).

STL components which are now part of the standard C++ library are defined in namespace std. We must therefore use the using namespace directive

using namespace std;

Components of STL

The STL contains several components. But at its core are three key components they are: -

  1. Containers

  2. Algorithms

  3. Iterators

Transforming a sequence: - The transform() algorithm applies a function to a range of clements & stores the outcome in result.

Containers: - Containers are objects that hold other objects, there are several different types, the vector class defines a dynamic array, dequeue creates a double ended queue, list provides a linear list. These containers are called sequence containers because in STL terminology a sequence is a linear list. In addition to the basic containers the STL also defines associative containers, which allow efficient retrieval of values based on keys.



Algorithms: - Algorithms are functions that can be used generally across a variety that can be used their contents. Although each container provides functions for its basic operations, STL provides more than sixty standard algorithm to support more extended on complex operations.


Iteration: - Iterators are objects that are more or less pointers. They give users the ability to cycle through the contents of a container in much the same way that users world use a pointer to cycle through an array.

Vector

The vector is the most widely used container. The vector class support a dynamic array. This is an array that can grow as needed. It stores elements in contiguous memory locations & enables direct access to any element using the subscript operator [ ]. A vector can change its size dynamically & therefore allocate memory as needed at run time.

Static Class Member: - Both function & data member of a class can be made static.

Static Data Member: - When users precede a member variables declaration whit static, users are telling the complier than only one copy of that variable will exist & that all objects of the class will share that variable. Unlike regular data members, individual copies of a static member variable are not made for each object.



Certain special characteristics of static Data Member

  1. It is initialized to zero when the first object of its class is created. No other initialization is permitted.

  2. Only one copy of the member is created for the entire class & is share by all the objects of that class, no matter how many objects are created.

  3. It is variable only within the class, but its life time is the entire program.


#include<iostream.h>

#include<conio.h>

class shared

{

public static int a,b;

public:

void set(int i, int j) {a=i; b=j}

void show();

};

int shared :: a; //define a

void shared :: show()

{

cout<<”\nThis is static a: “ <<a;

cout<<”\nThis is non-static b: “ <<b;

void main()

{

x.show();

y.set(2,2); //change a to 2

/*here a has been changed for both x and y because a is shared by both objects */

}

Virtual base Class: - A base class that has been qualified as virtual in the inheritance definition. In multiple inheritance, a derived class can inherit the member of base class via two or more inheritance paths. If the base class is not virtual, the derived class will inherit more than one copy of the member of the base class, however only one copy of its member will be inherited regardless of the of inheritance paths between base class & the derived class.



Inheritance


The mechanism of deriving a new class from an old one is called inheritance (or derivation). The old class is referred to as the base class & the new one is called the derived class or sub class.

Different class of inheritance: -

  1. Single Inheritance: - A derived class with only one base class is called single inheritance.

  2. Multiple Inheritance: - In this from of inheritance the derived class with several base classes.

  3. Hierarchical Inheritance: - The property of one class may be inherited by more than one class. This process is known as hierarchical inheritance.

  4. Multilevel Inheritance: - The mechanism of deriving a class from another ‘derived class’ is known as multilevel inheritance.
